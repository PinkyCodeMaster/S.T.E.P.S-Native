## 1. Purpose

Single source of truth to prevent scope creep, stop re-litigating decisions, preserve ethical intent, and guide planning. If a feature is not listed, it is not part of S.T.E.P.S.

## 2. How to Read

Each entry includes name, description, why it exists, S.T.E.P.S alignment, phase (MVP, Phase 2, 3, 4, or Parked/Not Planned), and status (Approved, Parked, Rejected).

## 12. Explicitly Rejected

- Credit score optimisation.
- Affiliate credit offers.
- Mandatory bank linking.
- Payment initiation.
- Automated enforcement features.
- Behavioural pressure mechanics.

## 13. Governance Rule

Any new feature must include justification, S.T.E.P.S alignment, ethical review, and phase assignment before it appears here. If it is not in this document, it does not ship.
